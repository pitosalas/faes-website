# Feature description for feature F20
## F20 — Extract inline HTML to Jinja2 templates
**Priority**: Medium
**Done:** yes
**Tests Written:** yes
**Test Passing:** yes
**Description**: Move all inline HTML out of Python source files into Jinja2 template files. The rule enforced is: no HTML or CSS block longer than 2 lines may live inside a Python f-string. Jinja2 is already a declared dependency; a new `templates/html/` directory holds all templates. `site_generator.py` gains a module-level `_TEMPLATES_DIR` path and a Jinja2 `Environment` (autoescape=False, trusted content). `content_loader.py` gains a module-level `_env` (autoescape=True, external file content). All public method signatures and generated HTML output are unchanged.

## Templates created
- `base.html` — full page shell: doctype, head, nav header, main, footer
- `year_chart.html` — Chart.js bar chart canvas + script block; uses `| tojson` for labels/values
- `grant_card.html` — single grant recipient card
- `grants_page.html` — grants page body: chart section + cards grid
- `person_card.html` — single person card with photo and bio
- `board_page.html` — board page body: board section + advisors section
- `content_page.html` — generic content page body wrapper
- `photo_figure.html` — photo shortcode `<figure>` element; autoescaping handles user content

## Design decisions
- `autoescape=False` in `site_generator.py` — HTML is generated by trusted Python code; pre-rendered HTML fragments passed via `| safe`
- `autoescape=True` in `content_loader.py` — `photo_figure.html` receives data parsed from external markdown files
- `| tojson` in `year_chart.html` replaces Python list repr to produce valid JS array literals
